# 画面構成・UIワイヤー

本ドキュメントは「麻雀 点数計算トレーニングアプリ」における画面構成およびUIワイヤーを定義するものであり、要件定義.md とは別資料として管理する。

---

## 共通UI方針

- PC・スマートフォンともに **横画面（ランドスケープ）前提**
- スマートフォン縦向き時は操作不可とし、横向き利用を促す案内を表示する
- 卓UIはフェーズに応じて表示範囲を切り替える
- 回答入力・正誤結果・解説フェーズでは、**卓UI全体は表示せず、自分の手牌のみを画面下部に固定表示**する

---

## 画面一覧

1. ホーム（スタート）画面  
2. 出題（卓表示）画面  
3. 回答入力画面  
4. 正誤結果画面  
5. 解説画面

---

## 1. ホーム（スタート）画面

### 目的
- 出題条件（ルール・難易度・回答方式）を設定し、ゲームを開始する

### 表示要素
- タイトル
- ルール選択（Mリーグ / 雀魂）
- 難易度選択（実用重視・デフォルト）
- 回答方式選択（テキスト入力 / 選択肢式6択）
- 開始ボタン

### レイアウト（簡易ワイヤー）

```
[ ルール選択 ]   [ 難易度選択 ]   [ 回答方式選択 ]

          [   開  始   ]
```

---

## 2. 出題（卓表示）画面

### 目的
- 実戦に近い情報配置で問題状況を提示する

### 表示要素
- 場風・局数
- 親表示
- ドラ表示牌
- 点数表示（4人分）
- 河（4人分）
- 自分の手牌（テンパイ状態）
- 演出スキップボタン

### 進行仕様
- 2〜6巡程度の進行演出後に和了イベントが発生
- スキップ操作時は **和了直前の状態まで遷移** する

### レイアウト（概念）

```
[ 上家 河 ]        [ 場風 / 局 / ドラ ]        [ 対面 河 ]

[ 左家 河 ]              卓中央                [ 右家 河 ]

────────────── 自分の手牌（下部・最大表示） ──────────────

                         [ スキップ ]
```

---

## 3. 回答入力画面

### 目的
- 点数計算の回答を入力する

### UI方針
- 卓UI全体は非表示
- **自分の手牌のみを画面下部に固定表示**
- 回答に必要な最小限の情報を画面上部に集約

### 表示要素（上部）
- 和了種別（ロン / ツモ）
- 和了者（親 / 子）
- ドラ表示牌
- 回答入力UI
- 「回答する」ボタン
- 「わからない」ボタン（不正解扱い）

### 入力UI仕様

- テキスト入力
  - ロン：入力欄1つ
  - ツモ：親の支払い入力欄 / 子の支払い入力欄
- 選択肢式
  - ロン：6択
  - ツモ：親＋子の支払いペアを6択

### レイアウト（簡易ワイヤー）

```
ロン / ツモ   親 / 子   ドラ：🀄

点数を入力してください
[        ]

[ 回答する ]   [ わからない ]

────────────────────────────────
自分の手牌（原寸・固定表示）
```

---

## 4. 正誤結果画面

### 目的
- 回答の正誤と正解を即時にフィードバックする

### 表示要素（上部）
- 正解 / 不正解表示
- 自分の回答
- 正解点数
- 「解説を見る」ボタン
- 「次の問題」ボタン

### レイアウト（簡易ワイヤー）

```
❌ 不正解

あなたの回答：5200
正解：ロン 3900

[ 解説を見る ]   [ 次の問題 ]

────────────────────────────────
自分の手牌（原寸・固定表示）
```

---

## 5. 解説画面（MVP）

### 目的
- 点数計算結果の理解を補助する

### 表示要素（上部）
- 翻数
- 符
- 最終点数（ロン / ツモの親・子）
- 「次の問題」ボタン

### レイアウト（簡易ワイヤー）

```
翻：3翻
符：40符

ロン：3900点

[ 次の問題 ]

────────────────────────────────
自分の手牌（原寸・固定表示）
```

### 将来拡張
- 役の内訳表示
- 符の内訳表示
- 詳細解説モーダル／別画面

---

## 画面遷移まとめ

- ホーム → 出題（卓表示）
- 出題 → 回答入力
- 回答入力
  - 回答する → 正誤結果
  - わからない → 不正解扱い → 正誤結果
- 正誤結果
  - 解説を見る → 解説
  - 次の問題 → 出題
- 解説 → 次の問題 → 出題

---

## レイアウト設計の一貫性

| フェーズ | 画面下部 | 画面上部 |
|--------|--------|--------|
| 出題 | 手牌＋卓UI | 卓情報 |
| 回答 | 手牌のみ | 入力UI |
| 正誤 | 手牌のみ | 正誤結果 |
| 解説 | 手牌のみ | 翻・符・点数 |
